<must_haves>

## Truths

- Place media sources are real media only: app-owned public photos or provider/open-media photos.
- AI-generated, stock-like, or illustrative media must not be representable as a successful place photo.
- Cache provider metadata and expiry by default; do not model durable copied provider binaries as the default path.
- Provider keys, provider headers, raw provider payloads, and private route context must not be stored.

</must_haves>

<tasks>

1. Add tests for the media cache contract first.
   - Create `tests/server/place-media-cache.test.mjs`.
   - Assert the source enum includes app, google, wikimedia, and foursquare or equivalent provider/open-media labels.
   - Assert no successful photo source can be `ai`, `generated`, `stock`, `illustrative`, or `missing`.
   - Assert cache metadata includes source, provider place id/reference, attribution, license/terms hint, expiry, match confidence, and failure code/state.
   - Assert tests fail if raw provider payload, prompt, provider headers, or secrets are represented in the cache shape.

2. Create the pure place media contract.
   - Add `lib/explore/place-media.ts`.
   - Define Zod schemas/types for `PlaceMediaSource`, `PlaceMediaCacheEntry`, `PlaceMediaResolutionInput`, `PlaceMediaResolutionResult`, and `PlaceMediaFailureCode`.
   - Model the result as a discriminated union such as `{ status: "photo", photo } | { status: "missing", reason }`.
   - Make source attribution and terms/license hints explicit enough for UI/API docs.

3. Add Drizzle schema for metadata cache.
   - Create `lib/db/schema/place-media-cache.ts`.
   - Include normalized place key, optional provider place id, optional provider photo reference, source, attribution, terms/license hint, expiresAt, failedAt, failureCode, matchConfidence, and timestamps.
   - Do not include API keys, provider headers, raw provider JSON, prompts, user-private diary text, or route prompt content.
   - Export from `lib/db/schema/index.ts`.

4. Add cache query helpers.
   - Create `lib/db/queries/place-media-cache.ts`.
   - Add helpers for get fresh entry, upsert photo metadata, record failure, and mark stale/expired.
   - Use existing Drizzle patterns and keep helpers small and testable.

5. Run focused verification and create summary.
   - Run `node scripts/run-node-tests.mjs tests/server/place-media-cache.test.mjs`.
   - Run existing focused tests: `tests/server/place-intelligence.test.mjs` and `tests/server/place-popup-renderer.test.mjs`.
   - Create `11-01-SUMMARY.md` with changed files, cache contract decisions, and verification results.

</tasks>
