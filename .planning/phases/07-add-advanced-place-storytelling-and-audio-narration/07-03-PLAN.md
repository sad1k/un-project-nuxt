<threat_model>
- False offline promise: browser caches can be evicted, so UI must verify saved audio before claiming availability.
- Privacy/cache scope: saved story audio must be scoped to the generated route/session/place story, not a global place cache.
- Auto-cache creep: do not cache every played/generated story; saving must be an explicit user action.
- Broad PWA scope creep: do not convert the route-generation notification service worker into a general offline media worker unless execution proves the client Cache API cannot satisfy the requirement.
- Offline mutation trap: do not attempt generation while offline; show unavailable if no saved audio exists.
- Requirement honesty: record `ADVPLACE-02` as partial/deferred in verification.
</threat_model>

<tasks>
1. Add offline behavior tests first.
   - Assert story audio is saved only through an explicit save action.
   - Assert cache keys include route/session/variant/routePoint identity or an authenticated story audio URL bound to that identity.
   - Assert offline unsaved playback shows unavailable and does not call generation.
   - Assert saved audio can be removed and the UI returns to not-saved state.
   - Assert no broad service-worker media caching is introduced unless deliberately justified.

2. Implement explicit Cache API support in the story composable.
   - Use `CacheStorage.open("wanderlog-place-story-audio-v1")` for generated story audio.
   - Implement `saveOffline`, `removeOffline`, and `checkOfflineAvailability` around the authenticated story audio URL/cache key.
   - Verify audio fetch responses are OK and audio-like before caching.
   - Prefer cached audio when offline and saved; otherwise use the authenticated online audio URL.
   - Treat cache miss, unsupported Cache API, or browser eviction as recoverable states.

3. Add story card offline states and controls.
   - Add simple states: not saved offline, saving, available offline, remove offline, unavailable offline.
   - Keep controls compact and secondary to playback. Do not add a storage manager, queue, or global downloads view.
   - Surface clear failure text when offline playback is unavailable because the story was not saved.

4. Avoid unnecessary service-worker expansion.
   - Inspect `public/route-generation-sw.js` and leave it focused on route-generation notifications unless explicit Cache API playback cannot work.
