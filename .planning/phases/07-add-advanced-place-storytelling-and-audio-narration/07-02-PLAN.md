<threat_model>
- Surprise audio: generation and playback must happen only after an explicit user tap.
- Surface creep: the popup is only a small CTA; the player belongs in the route sidebar.
- Unsupported transcript: do not add transcript/story text, captions, or a long reading view.
- `ADVPLACE-02` drift: do not add voice/language selectors or claim full voice/language choice support.
- Client secret leak: the client composable must call app endpoints only and must never receive provider keys.
- Inaccessible controls: basic play/pause/replay/error/loading controls must be keyboard and screen-reader understandable.
</threat_model>

<tasks>
1. Add UI/source tests before implementation.
   - Assert `place-popup.ts` exposes only a compact "Listen to story" CTA and does not render an audio player.
   - Assert the route panel/story card owns player states and source/status cues.
   - Assert there is no transcript/story body, no voice selector, and no language selector in Phase 7 UI.
   - Assert audio generation/playback paths are user-triggered rather than automatic on selection.

2. Implement the story client composable.
   - Add `composables/use-place-story.ts` for story status loading, generation requests, playback state, progress, replay, loading, error, and cleanup.
   - Use `Audio()`/`HTMLAudioElement` on the client side only, with event listeners cleaned up when the selected story changes or the component unmounts.
   - Keep the composable route-point scoped, mirroring the cache discipline in `use-place-intelligence.ts`.
   - Use the authenticated app endpoints from Plan 07-01; do not call third-party audio APIs from the browser.

3. Build the route-sidebar story card.
   - Add `components/explore/place-story-card.vue` and embed it in `components/explore/route-panel.vue` for the selected/generated route point.
   - Render compact states: unavailable, idle, generating/loading, playing, paused, replay-ready, and failed.
   - Show a small source/status note and AI-voice disclosure.
   - Keep text tight and appropriate for a sidebar panel. Do not add an explanatory feature tutorial.

4. Wire popup CTA to sidebar focus.
   - Update `components/explore/place-popup.ts` to render a minimal "Listen to story" action only when the popup represents a generated route point that can be associated with story status.
   - Wire popup/marker callbacks through existing route marker/mapbox integration so tapping the CTA selects or focuses the route point in the route panel.
   - Avoid adding full playback controls inside Mapbox popup HTML.

5. Integrate selected-place state in Explore.
   - If the existing selected route point state is insufficient, add the smallest route-point selection/focus state in `pages/explore.vue` and pass it to route panel/marker wiring.
   - Preserve existing map route interactions, diary save state, route history, weather tips, and place intelligence behavior.

6. Verify and summarize.
   - Run focused story UI/source tests.
   - Run `pnpm lint:source`.
   - Use browser/manual verification if a local app target is already practical: select a generated place, tap the popup CTA, trigger story generation/playback, pause/replay, and confirm unavailable/error states.
   - Create `07-02-SUMMARY.md` with changed files, verification evidence, and offline-plan handoff notes.
