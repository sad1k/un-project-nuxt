| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-051-01 | Browser disconnect cancels provider work before terminal status is saved. | high | Move provider loop into a server runner started after persisted session/variant creation; do not depend on response reader lifetime. |
| T-051-02 | Duplicate background runs corrupt route events or points. | high | Add generation claim/heartbeat helpers and single-active-run guard per variant. |
| T-051-03 | Failed background work leaves permanent stale `generating` rows. | medium | Persist heartbeat and failure/terminal metadata; add stale detection plan in Wave 4. |
| T-051-04 | Provider diagnostics leak secrets through lifecycle logs. | high | Reuse sanitized error categories and never persist provider headers, prompts, or raw personal context. |
</threat_model>
