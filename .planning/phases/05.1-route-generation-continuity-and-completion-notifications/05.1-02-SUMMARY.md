# Phase 5.1-02 Summary: Route Session Status and History UI

## Completed

- Added `server/api/ai/route-sessions.get.ts` for authenticated, compact route session summaries.
- Added stale-generation detection using variant heartbeat age and exposed `displayStatus` so the UI does not show stale work as an endless active spinner.
- Added `composables/use-route-generation-status.ts` for shared polling outside Explore.
- Added `components/app/route-generation-indicator.vue` and mounted it in the app nav plus Explore overlay.
- Extended Explore restore support so `/explore?sessionId=<id>` opens a saved/completed session.
- Extended route history to show recent saved generations, not only variants for the currently loaded session.
