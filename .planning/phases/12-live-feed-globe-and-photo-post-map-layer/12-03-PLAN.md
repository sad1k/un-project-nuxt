## Truths

- D-07: Live reads stay public because the payload is safe and intentionally public.
- D-09/D-15: The stream must not expose any fields absent from the safe public globe serializer.
- D-10: New public photo posts should appear online/near-real-time as animated points.
- D-11/D-12/D-13: Live arrivals must pass through the same density limiter as initial page load.
- No new runtime dependency should be added for live updates unless an implementation blocker proves it necessary.

</must_haves>

<tasks>

1. Add live update tests first.
   - Create `tests/server/feed-globe-live.test.mjs`.
   - Assert the stream endpoint is public and emits only serialized public globe post events.
   - Assert private/non-public/hidden-moderation posts are not emitted.
   - Assert a client-side live arrival passes through the density helper and replaces/fades the oldest visible point when a bucket is full.
   - Assert polling fallback can merge newer posts without duplicating existing points.

2. Add public SSE endpoint.
   - Create `server/api/public/feed-globe/stream.get.ts`.
   - Prefer H3/Nitro event-stream helpers if available in the app runtime.
   - Emit the same safe `PublicFeedGlobePost` payload used by `GET /api/public/feed-globe`.
   - Support a lightweight `since` or cursor strategy so reconnects can catch up without replaying the whole globe.
   - Keep connection cleanup explicit.
   - Do not log raw payloads or private fields.

3. Add polling fallback and merge behavior.
   - Extend `composables/use-feed-globe.ts`.
   - Use `EventSource` on the client when available.
   - Fall back to periodic `GET /api/public/feed-globe?since=...` polling when SSE fails or is unavailable.
   - Deduplicate by post id.
   - Apply the density helper after every initial load, stream event, and polling refresh.
   - Expose enough state for the globe component to animate new/fading points.

4. Refine globe animation behavior.
   - Update `components/feed/feed-globe.client.vue`.
   - Add subtle arrival/fade states for new and replaced points.
   - Keep animation bounded and avoid layout shift.
   - Preserve popup-safe content and cleanup logic.

5. Document API contract.
   - Update `lib/api-docs/openapi.ts`.
   - Document `GET /api/public/feed-globe` and live stream behavior.
   - Mark public endpoints with no auth requirement.
   - Document safe payload fields and privacy exclusions.
   - Update `tests/server/api-docs.test.mjs`.

6. Run final verification.
   - Run all Phase 12 focused tests.
   - Run `npm run lint:source` if feasible.
   - Run `npm run typecheck` if feasible; if existing unrelated blockers remain, record exact blockers.
   - Run targeted secret/private-field scans on changed Phase 12 source.
   - Perform browser smoke verification for `/feed` list/globe tabs, live arrival/polling fallback where practical, popup content, guest visibility, and mobile/desktop layout.

7. Create final artifacts.
   - Create `12-03-SUMMARY.md`.
   - Create `12-VERIFICATION.md` mapping `LIVEGLOBE-01` through `LIVEGLOBE-06` to evidence, commands, browser notes, known risks, and release blockers.

</tasks>
