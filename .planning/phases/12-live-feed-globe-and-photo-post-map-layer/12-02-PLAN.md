## Truths

- D-02: The feed keeps its normal list and gains a distinct global globe tab/surface.
- D-04: Users should see a clear quick action for publishing uploaded photos to the feed.
- D-07: Guests can open the globe tab.
- D-10: Points should feel alive, with animated arrival/fade states where practical.
- D-11/D-12/D-13: A local radius shows no more than 3-4 active points; newer points replace/fade the oldest visible point; hidden posts produce `+N`.
- D-14/D-15: Popups show only photo, place, author display name, and date.

</must_haves>

<tasks>

1. Add density tests first.
   - Create `tests/server/feed-globe-density.test.mjs`.
   - Assert the pure limiter keeps the newest 4 visible points per local bucket.
   - Assert a new point in a full bucket removes or marks the oldest visible point.
   - Assert hidden overflow count is exposed as `+N` data.
   - Assert points in separate buckets remain independently visible.
   - Assert the function is deterministic and independent from Mapbox/browser globals.

2. Implement pure globe density helper.
   - Create `lib/feed/globe-density.ts`.
   - Define typed point input and render output shapes.
   - Use a documented local-radius bucket strategy that is simple and stable enough for tests.
   - Keep max visible per bucket configurable with default `4`.
   - Return visible points, fading/removed point ids if needed, and overflow cluster indicators.

3. Add feed globe data composable.
   - Create `composables/use-feed-globe.ts`.
   - Fetch `GET /api/public/feed-globe`.
   - Normalize API rows into density-helper input.
   - Expose loading, error, refresh, visible points, overflow indicators, and selected popup state.
   - Keep the composable client-safe and do not import server-only modules.

4. Build Mapbox globe component.
   - Create `components/feed/feed-globe.client.vue`.
   - Initialize a dedicated Mapbox instance for the feed globe instead of sharing Explore route-map singleton state.
   - Use globe projection, fog/atmosphere, low-zoom spin, and responsive full-width sizing.
   - Render visible posts as animated point layers or markers and render overflow indicators as compact `+N` markers/layers.
   - Use safe popup HTML/components with photo, place, author display name, and date only.
   - Clean up map, layers, sources, timers, and popup listeners on unmount.

5. Wire feed tab and quick publish affordance.
   - Update `pages/feed.vue` so the tab model has a real feed/list tab and globe tab.
   - Keep existing list feed as the default.
   - Render the globe client component only when the globe tab is active or lazy-loaded.
   - Keep the publish action prominent for authenticated users and route to existing quick publish flow.
   - For guests, keep reading available and show authentication-only affordance for publishing without blocking the globe.

6. Enrich feed cards where safe.
   - Update `stores/feed.ts` and `components/feed/post-card.vue` only as needed for safe public place labels and quick post context.
   - Replace hardcoded "trip place" text with safe public place name when available.
   - Do not display private diary descriptions or hidden route context.

7. Add UI regression tests.
   - Create `tests/server/feed-globe-ui.test.mjs` with render/string-level tests for tab state, popup-safe content, and guest visibility where the existing test harness supports it.
   - Assert forbidden private fields are not rendered in globe popup/card helpers.

8. Verify and summarize.
   - Run density, UI, and Phase 12 API tests.
   - Use browser verification if a dev server is practical: feed list tab, globe tab, guest read, authenticated publish affordance, dense point overflow.
   - Create `12-02-SUMMARY.md` with UI decisions and verification results.

</tasks>
