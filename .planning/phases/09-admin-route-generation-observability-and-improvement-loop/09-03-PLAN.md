<tasks>
1. Add UI source tests first.
   - Create `tests/server/admin-route-generation-ui.test.mjs`.
   - Assert admin pages call `/api/admin/route-generations`.
   - Assert list page renders filters/status/stage/code/retryability/timings without route point details.
   - Assert detail page renders sanitized route snapshot fields.
   - Assert navigation entry is gated on admin role.
   - Assert no role-management UI is introduced.

2. Build admin route-generation list page.
   - Create `pages/admin/route-generations.vue`.
   - Use a dense operational layout with filters for status, failure stage, failure code, provider/model, date range or recent window, and stale/failed state where supported.
   - Show metadata and sanitized request summary only: ids, safe user identifier, city, days/interests summary, status/display status, point count, timings, retry count, diary status, notification status, failure stage/code, retryability, safe explanation.
   - Link each row to the detail page.
   - Keep copy concise and operational.

3. Build admin route-generation detail page.
   - Create `pages/admin/route-generations/[session-id].vue`.
   - Fetch the admin detail endpoint.
   - Show sanitized route snapshot: title, summary, generated point names, coordinates, day/sequence, confidence, approximate distance, timing/count fields.
   - Show safe timeline/events only from allowlisted fields.
   - Do not render raw prompts, raw model output, raw provider body, raw `requestContextJson`, raw `payloadJson`, or private diary/location context.

4. Add admin navigation entry.
   - Add an admin-only entry in the least noisy existing navigation surface, likely user menu or side rail.
   - Render it only when `authStore.user?.role === "admin"`.
   - Do not add admin links to the mobile toolbar unless the UI remains uncluttered and role-gated.
   - Keep server-side checks authoritative.

5. Run verification and create Phase 9 verification artifact.
   - Run focused admin tests.
   - Run `pnpm test:server`.
   - Run `pnpm lint:source`.
   - Run `pnpm typecheck` and `pnpm build`, or record known unrelated blockers if they still fail.
   - Manually verify with an admin-role account: admin list loads, filters work enough for v1, detail opens intentionally, non-admin access is denied, and raw diagnostics are not visible.
   - Create `09-VERIFICATION.md` mapping decisions D-01 through D-16 to evidence.
</tasks>
