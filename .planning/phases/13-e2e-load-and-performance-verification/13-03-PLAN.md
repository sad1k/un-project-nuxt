<tasks>

1. Add failing tests for reporting and cleanup contracts.
   - Extend `tests/server/load-runner-source.test.mjs`.
   - Assert reports include p50/p95/p99, RPS, status codes, errors, timeout counts, read/write p95, threshold pass/fail, created photo/post totals, and run id.
   - Assert cleanup is explicit and run-id based.
   - Assert docs mention preserving data by default and separate cleanup.
   - Assert package scripts expose the full e2e load run and cleanup commands.

2. Add durable report output.
   - Update `tests/load/run-load.mjs` and metrics helpers to write JSON and readable markdown or console summaries under the load output directory.
   - Include per-step, per-class, and total metrics.
   - Include created counts for users, photos, posts, places/logs, and S3 objects.
   - Include threshold evaluation for read p95 <= 800ms, write p95 <= 1500ms, error rate <= 1%, and timeout rate <= 0.5%.
   - Avoid writing raw cookies, session tokens, CSRF tokens, full signed URLs, or secrets.

3. Implement explicit cleanup.
   - Create `tests/load/cleanup-run.mjs`.
   - Accept `--run-id` and optional manifest path.
   - Delete run-marked posts, images, location logs, locations, sessions, and users from the local DB in dependency-safe order.
   - Delete S3 objects listed in the manifest on a best-effort basis.
   - Refuse cleanup without a run id.
   - Print a dry-run summary before destructive cleanup unless a force flag is present.

4. Add package scripts.
   - Update `package.json` with scripts such as `load:e2e`, `load:e2e:dry-run`, and `load:cleanup`.
   - Preserve existing scripts.
   - Keep scripts explicit about storage opt-in.

5. Update load documentation.
   - Update `tests/load/README.md`.
   - Document Phase 13 prerequisites: local app, local DB URL, Better Auth seeded sessions, CSRF handling, S3-compatible storage configuration, and storage opt-in.
   - Document the full default command, smoke command, thresholds, report paths, manifest path, and cleanup command.
   - Document that data is preserved after runs for diagnostics.
   - Document that results are a local baseline, not production capacity.

6. Run final Phase 13 verification.
   - Run focused source tests.
   - Run existing load smoke/dry-run commands.
   - Run `npm run load:e2e:dry-run`.
   - If local app/storage are available, run a tiny e2e opt-in smoke and then optionally the full 10-minute profile.
   - Run targeted lint on changed Phase 13 files.
   - Run `npm run lint:source`, `npm run typecheck`, and `npm run build` if feasible; record existing unrelated blockers instead of hiding them.
   - Run secret scans for docs, manifests, reports, and changed load scripts.

7. Create final artifacts.
   - Create `13-03-SUMMARY.md`.
   - Create `13-VERIFICATION.md` with commands, results, performance thresholds, any run ids created, cleanup status or preserved-data note, known risks, and release blockers.

</tasks>
