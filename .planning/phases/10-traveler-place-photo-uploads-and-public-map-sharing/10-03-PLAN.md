<tasks>
1. Add UI and privacy tests first.
   - Create `tests/server/public-place-photo-ui.test.mjs`.
   - Assert public photo layer/component fetches the public endpoint, not owner endpoints.
   - Assert public popup/card renders photo, place, author display name, and date only.
   - Assert UI source does not render private diary description as public caption.
   - Assert owner controls call the authenticated visibility endpoint.
   - Assert public photo map layer is distinct from personal diary markers by component name/test id/classes.
   - Assert no full moderation/report/admin queue UI appears in Phase 10.

2. Build public photo layer.
   - Add `components/place-photo/public-photo-layer.client.vue` or equivalent.
   - Fetch `/api/public/place-photos` with map bounds/pagination where available.
   - Render distinct photo markers or first-slice bounded markers; add clustering only if it fits the established map library without new dependencies.
   - Keep the layer usable on public/unauthenticated pages.
   - Do not mix public photo points into private `mapStore.mapPoints` unless the visual distinction is explicit and tested.

3. Build public photo popup/card.
   - Add `components/place-photo/public-photo-popup.vue`.
   - Render only image, public place name, author display name, and published date.
   - Do not render captions, diary descriptions, private notes, internal moderation labels, raw ids beyond stable public id, or owner-only actions.
   - Use existing S3 public bucket URL pattern only if safe for public images.

4. Integrate public layer into map/place surfaces.
   - Add the public layer to the first appropriate map surface, likely Explore map or a dashboard/public map surface chosen by implementation evidence.
   - Provide a small layer toggle only if needed to avoid clutter.
   - Preserve existing route map/diary marker behavior.
   - Make loading/empty/error states compact and non-marketing.

5. Add owner visibility controls.
   - Add `components/place-photo/photo-visibility-controls.vue`.
   - Surface controls near owner's diary image/detail view after private save.
   - Make "Make public" and "Make private" explicit actions.
   - Show public status and published/hidden/private state without exposing admin moderation tooling.
   - Ensure owner unpublish keeps the diary image visible to the owner.

6. Run full Phase 10 verification.
   - Run focused Phase 10 tests.
   - Run `pnpm test:server`.
   - Run `pnpm lint:source`.
   - Run `pnpm typecheck` and `pnpm build`, or record known unrelated blockers.
   - Manually verify authenticated capture/save/publish/unpublish and unauthenticated public photo visibility.
   - Create `10-VERIFICATION.md` mapping PHOTO-01 through PHOTO-06 and D-01 through D-25 to evidence or documented blockers.
   - Create `10-03-SUMMARY.md`.
</tasks>
