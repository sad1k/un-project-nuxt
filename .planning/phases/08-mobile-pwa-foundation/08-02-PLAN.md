<threat_model>
- Private-data caching: service-worker fetch handlers must bypass `/api/**`, Better Auth routes, S3/object URLs, map/provider tiles, and generated story audio.
- Notification regression: migrating registration from `/route-generation-sw.js` must preserve push payload parsing, notification tags, click navigation, and authenticated subscription persistence.
- Offline overclaiming: offline fallback must clearly provide a shell/page fallback only and must not imply diary edits, route generation, AI, maps, or provider data work offline.
- Service-worker conflicts: avoid registering multiple workers with overlapping scopes.
</threat_model>

<tasks>
1. Add service-worker boundary tests first.
   - Create `tests/server/pwa-service-worker.test.mjs`.
   - Assert `public/wanderlog-sw.js` handles `install`, `activate`, `fetch`, `push`, and `notificationclick`.
   - Assert fetch caching excludes `/api/`, auth/session paths, map/provider tile hosts or URL patterns, S3/object paths, and place-story audio.
   - Assert route notification registration uses the new service worker path.
   - Update existing notification/offline tests so they protect the new unified boundary.

2. Add the unified app service worker.
   - Create `public/wanderlog-sw.js`.
   - Precache only a small shell set such as `/offline.html`, `/manifest.webmanifest`, and local icon/favicon assets.
   - Add navigation-only fallback behavior for HTML document requests when the network is unavailable.
   - Retain route-generation `push` and `notificationclick` handling from `route-generation-sw.js`.
   - Use cache version constants and cleanup old app-shell caches in `activate`.

3. Add an offline fallback document.
   - Create `public/offline.html` with concise, static copy that does not describe unsupported offline editing/generation.
   - Keep it dependency-free and safe to serve without authentication.

4. Migrate route-generation notification registration.
   - Update `composables/use-route-generation-notifications.ts` to register `/wanderlog-sw.js`.
   - Either leave `public/route-generation-sw.js` as a compatibility shim or remove/deprecate it only if tests confirm there is one intended service-worker path.
   - Preserve VAPID public-key gating and authenticated subscription POST behavior.

5. Verify and summarize.
   - Run `node scripts/run-node-tests.mjs tests/server/pwa-service-worker.test.mjs tests/server/route-generation-notifications.test.mjs tests/server/place-story-offline.test.mjs`.
   - Run `pnpm lint:source`.
   - Create `08-02-SUMMARY.md` with changed files, verification evidence, and any manual unregister/reload notes for browser testing.
</tasks>
