<threat_model>
- Credential exposure: manifest, icon files, and app head metadata must not include provider tokens, S3 URLs, route context, user IDs, or secret-bearing config.
- Scope creep: installability does not mean native mobile app work, offline writes, or broad private-data caching.
- False installability claims: plan must add tests that prove manifest link, core manifest members, and icon references exist before claiming completion.
- New dependency drift: do not add `@vite-pwa/nuxt`, Workbox, or any other PWA dependency in this plan.
</threat_model>

<tasks>
1. Lock installability expectations with source tests.
   - Create `tests/server/pwa-installability.test.mjs`.
   - Assert `app.vue` or Nuxt app head configuration links `/manifest.webmanifest`.
   - Assert manifest contains `name`, `short_name`, `id`, `start_url`, `scope`, `display`, `theme_color`, `background_color`, and multiple icon entries.
   - Assert manifest/icon content does not reference server-only env names or private route/session data.

2. Add the static web app manifest.
   - Create `public/manifest.webmanifest`.
   - Use stable app identity for WanderLog.
   - Set `start_url` and `scope` to `/`.
   - Prefer `display: "standalone"` and include a conservative theme/background color aligned with the existing app shell.
   - Include regular and maskable icons.

3. Add install icons without provider or user data.
   - Add `public/icons/wanderlog-icon.svg` and `public/icons/wanderlog-maskable.svg`.
   - Keep the graphics simple, deterministic, and source-controlled.
   - Do not generate external image URLs.

4. Register manifest and mobile metadata.
   - Add app-level `useHead` or Nuxt head metadata so every route exposes the manifest link.
   - Add `theme-color`, `apple-mobile-web-app-capable`, and useful mobile viewport-related metadata only if they do not conflict with existing Nuxt defaults.
   - Keep changes in the app shell, not feature pages.

5. Verify and summarize.
   - Run `node scripts/run-node-tests.mjs tests/server/pwa-installability.test.mjs`.
   - Run `pnpm lint:source`.
   - Create `08-01-SUMMARY.md` with changed files, verification evidence, and handoff notes for the service-worker plan.
</tasks>
